1. Overview

IcefaceTDN Beat Machine is a kinetic music instrument. Notes are not triggered by pressing keys — they are triggered by movement. The distance your cursor travels across the canvas determines when the next note fires.

The instrument is built around a single theoretical principle: T(D,N), a dual transformation that simultaneously alters both the timing and pitch of non-diatonic notes based on a single parameter, N.

At N = 1.0, the instrument behaves like standard equal temperament — all notes are equal in both pitch and timing. As N decreases toward 0, non-diatonic notes (black keys) become both shorter in duration and closer in pitch to their diatonic neighbors.

2. How to Play

Auto Play

Click the ▶ PLAY button in the sidebar. The cursor follows a geometric path automatically. Select the path shape from the dropdown next to the Play button. Adjust Speed to control how fast the cursor moves.

Manual Play

Click and drag anywhere on the canvas. Notes trigger as you move. The faster you move, the more notes fire in a given time. Releasing the mouse stops playback. Manual dragging will stop Auto Play.

3. T(D,N) — The Core Principle

T(D,N) stands for Temporal-Diatonic transformation with parameter N. It is the defining characteristic of this instrument — a single slider that simultaneously controls two musical dimensions.

The Diatonic Reference Set D

D is the set of pitch classes corresponding to the white keys of a standard keyboard:

D = { 0, 2, 4, 5, 7, 9, 11 } → C, D, E, F, G, A, B

This is a topological reference, not a tonal scale. It defines which notes are diatonic (inside D) and which are non-diatonic (outside D — the black keys).

The N Parameter

N ranges from 0 to 1. It controls two things simultaneously for every non-diatonic note:

DimensionDiatonic noteNon-diatonic note
Time (step distance)full stepSizestepSize × N
Pitch (cents offset)0¢+ (1 − N) × 100¢
Example — N = 0.5, Iceface Tuning:
A C# (non-diatonic) will fire after half the normal movement distance, and its pitch will be shifted +50 cents toward D. At the same time, the Iceface tuning preset shifts C# by +50¢ from its equal-tempered position.

Both transformations converge at the same value — the pitch and time compression are unified by a single N.

T(D,N) Mode vs T(D,N) LEAP Mode

T(D,N) — The original transformation. Step timing is determined by the identity of the current note (is it in D or not?).

T(D,N) LEAP — An extended mode. Step timing is determined by the distance (in semitones) between the current note and the previous note. Larger intervals get longer durations; adjacent notes move quickly.

T(D,N) mode: diatonic note → stepSize × 1.0 non-diatonic → stepSize × N T(D,N) LEAP mode: interval = |current − previous| semitones stretch = 1.0 + (interval − 1) / 10 → 1st: 1.0×, 11th: 2.0× step = stepSize × (1 + (stretch−1) × N)

In LEAP mode, N scales the degree of stretch. At N = 0, all steps are equal regardless of interval. At N = 1.0, large leaps breathe naturally while stepwise motion flows quickly — a principle common to expressive melodic performance.

4. Scale & Tuning

Scale

Defines which pitch classes are available as note sources. The selected scale, combined with the Base note and Octave spread, determines the full note array that the kinetic engine steps through.

Tuning

Applies microtonal offsets (in cents) to each pitch class, modifying intonation without changing the note names. When an ICEFACE tuning is selected, N automatically sets to 0.5 and the scale switches to Chromatic, enabling the full T(D,N) transformation.

Non-Iceface tunings (Equal, Just, Meantone, etc.) reset N to 1.0 and the scale to Major.

Base Note

The lowest note of the generated scale. The full note array spans upward from this note for the number of octaves specified.

Octaves

The number of octaves the note array covers. A wider spread gives a larger range and more variety in melodic movement.

5. Synth

Wave

The oscillator waveform. Sine is the purest tone. Triangle is similar but slightly brighter. Square has strong odd harmonics — hollow and reedy. Sawtooth is the richest, with all harmonics present — the classic TB-303 basis.

Cutoff

Low-pass filter cutoff frequency. Lower values darken the tone; higher values open up the full harmonic content. The filter responds in real time as you adjust.

Resonance

Emphasis at the cutoff frequency. Higher values produce a pronounced peak — the characteristic "wah" of acid bass sounds. Extreme values can produce self-oscillation.

Glide (Portamento)

Smooths pitch transitions between notes. At 0, notes change instantly. As Glide increases, the oscillator slides from the previous pitch to the new one. Creates legato and TB-303-style portamento.

Octave

Shifts the entire note array up or down by octaves. −1 or −2 produces bass register sounds appropriate for bass lines and sub-bass.

Volume

The output level of the synthesizer voice.

6. Drums

The drum engine synthesizes three voices from scratch using the Web Audio API — no samples are used.

VoiceSynthesis
K — KickPitch-swept sine wave (160Hz → 45Hz) with click transient. 0.4s decay.
S — SnareSine body + noise + snap oscillator + attack transient. 0.35s decay.
H — Hi-hatFiltered white noise with fast exponential decay. 0.08s.

Drum steps advance in sync with note triggers — each time the kinetic engine fires a note, the drum sequencer advances one step.

Volume

Master level for all drum voices.

Filter

Low-pass filter applied to the drum bus independently of the synth. Useful for dialing in the drum tone without affecting the melodic content.

7. Direction

Controls how the note index advances through the note array after each trigger.

ModeBehavior
UPSequential ascending. Wraps from top back to bottom.
DOWNSequential descending. Wraps from bottom back to top.
BOUNCEAscends to the top of the array, then reverses and descends, continuously.
RANDOMEach trigger picks a random note from the array. Combined with T(D,N) LEAP, produces interval-proportional timing even for unpredictable sequences.

8. Kinetic Parameters

Step

The base distance threshold (in canvas-fraction units) that the cursor must travel before triggering the next note. Smaller values trigger notes more rapidly for the same cursor speed. This value is modified by T(D,N) for non-diatonic notes.

Speed

Only active during Auto Play. Controls how fast the cursor moves along the geometric path. At Speed = 60, the cursor moves at a moderate tempo. Increase for faster, denser playback; decrease for slower, spacious phrasing.

9. Drum Pattern Grid

The drum pattern editor at the bottom of the screen shows three rows — K (Kick), S (Snare), H (Hi-hat) — across all pattern steps.

Editing

Click any cell to toggle it on or off. An active cell is highlighted in color. The currently playing step is marked with a green border.

Controls

  • Pattern dropdown — Load a built-in pattern.
  • RANDOM — Generate a random pattern. Kick: 30% probability per step, Snare: 20%, Hi-hat: 40%.
  • CLEAR — Silence all steps.

Pattern Length

Pattern length varies by preset (12 or 16 steps). The drum sequencer loops independently of the note array length, creating polyrhythmic relationships when the lengths differ.

Note on timing: Drum steps and note triggers are tied together — each note trigger advances the drum sequencer by one step. This means the drum pattern tempo is entirely determined by your cursor movement speed and the kinetic step size, not by a clock. This is fundamental to T(D,N): time is spatial, not metronomic.
